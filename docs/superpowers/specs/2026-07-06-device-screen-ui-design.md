# ESP32-P4 设备屏幕本地 UI 设计

## 目标

在 `ESP32-P4-WIFI6-Touch-LCD-4.3` 自带触摸屏上增加本地操作界面，让现场用户不依赖 PC 后台也能完成基础使用：

- 扫描附近 Wi-Fi，点选 SSID，输入密码并连接网络。
- 在屏幕上查看当前 OV5647 摄像头画面。
- 本地手动触发浇灌 IO，且只能限时开启，到时自动关闭。

该功能必须保留现有 PC 后台能力：设备 MQTT 上线、后台 MJPEG 预览、远程限时浇灌继续可用。

## 页面结构

屏幕采用 LVGL 本地 UI，第一版使用底部或顶部三页切换：

1. `网络`
   - 显示当前 Wi-Fi 状态：未连接、连接中、已连接。
   - 显示当前 SSID、IP 地址、MQTT 状态。
   - 提供 `扫描 Wi-Fi` 按钮。
   - 扫描完成后显示 Wi-Fi 列表，用户点选 SSID。
   - 点选后弹出密码输入框和 LVGL 屏幕键盘。
   - 点击 `连接` 后保存 `wifi_ssid` 和 `wifi_pass` 到 NVS，并发起重连。

2. `摄像头`
   - 显示当前 OV5647 实时画面。
   - 屏幕画面和 HTTP MJPEG 预览共享同一个摄像头采集源。
   - 页面顶部显示设备在线状态、IP 地址和浇灌状态。
   - 如果摄像头初始化失败，显示中文错误状态，不阻塞 Wi-Fi 和浇灌功能。

3. `浇灌`
   - 显示当前 IO 状态：关闭、运行中。
   - 提供固定时长按钮：`10秒`、`30秒`、`60秒`。
   - 提供自定义秒数输入，范围 `1-900` 秒。
   - 点击后调用现有 `nursery_irrigation_start_limited()`。
   - 运行中显示倒计时和 `立即关闭` 按钮。
   - 任何重启、Wi-Fi 断开、异常路径仍默认关闭 IO。

## 架构

新增固件模块：

- `nursery_ui.h/.c`
  - 初始化 BSP 显示、触摸、LVGL。
  - 创建三个中文页面。
  - 接收网络、MQTT、浇灌、摄像头状态更新。

- `nursery_wifi.h/.c`
  - 从 `app_main.c` 中拆出 Wi-Fi 初始化、扫描、连接、事件处理。
  - 暴露 `nursery_wifi_scan()`, `nursery_wifi_connect(ssid, password)`, `nursery_wifi_status()`。
  - 保存 Wi-Fi 配置到 NVS。

扩展现有模块：

- `nursery_camera`
  - 增加屏幕预览回调或帧获取接口。
  - 保持 HTTP MJPEG 推流继续工作。
  - 摄像头采集缓冲仍按 128 字节对齐，避免 CSI DMA 断言。

- `nursery_irrigation`
  - 增加剩余秒数查询接口，用于屏幕倒计时。
  - 保持所有开启路径都必须带时长。

## 摄像头显示策略

优先沿用 Waveshare `09_video_lcd_display` 示例方式：

- 使用 BSP 启动 MIPI-DSI LCD 和触摸。
- 使用 PPA 将摄像头 RGB565 帧缩放/裁切到 LCD 缓冲。
- 摄像头采集任务快速归还 CSI 缓冲，避免长时间阻塞 DMA。

HTTP MJPEG 编码仍使用复制出的帧，避免 JPEG 编码占用 CSI 驱动缓冲。

如果屏幕预览和 MJPEG 同时运行时性能不足，第一版以稳定优先：

- 屏幕预览保持实时。
- MJPEG 降低帧率到约 3-5 fps。
- 不牺牲浇灌安全关闭逻辑。

## Wi-Fi 配网细节

Wi-Fi 扫描采用 STA 模式扫描：

- 列表显示 SSID、信号强度、加密状态。
- 隐藏空 SSID。
- 同名 SSID 合并或只显示信号最强项。
- 密码不在屏幕上明文长期显示，输入框默认密码模式。
- 连接失败时显示中文错误，如 `密码错误或信号弱`。

保存成功后写入 NVS namespace `nursery`：

- `wifi_ssid`
- `wifi_pass`

下一次启动直接读取 NVS 自动连接。

## 安全策略

- 浇灌 IO 上电默认关闭。
- 本地屏幕、远程 MQTT 都只能限时开启。
- 最大浇灌时长仍为 900 秒。
- Wi-Fi 断开时执行 `nursery_irrigation_safe_off("wifi disconnected")`。
- 重启、异常、命令失败时保持关闭。
- 本地 `立即关闭` 直接调用 `nursery_irrigation_safe_off("local screen stop")`。

## 中文界面要求

开发板屏幕上的所有可见文本使用中文：

- 页面：`网络`、`摄像头`、`浇灌`
- 操作：`扫描 Wi-Fi`、`连接`、`取消`、`立即关闭`
- 状态：`未连接`、`连接中`、`已连接`、`离线`、`运行中`、`已关闭`
- 错误：`摄像头初始化失败`、`Wi-Fi 连接失败`、`请输入密码`

## 测试计划

固件验证：

- 编译通过：`idf.py build`
- 烧录到 COM4：`idf.py -p COM4 flash`
- 串口确认无重启、无 `assert failed`
- 屏幕正常点亮，触摸可用

Wi-Fi 验收：

- 点击 `扫描 Wi-Fi` 能看到附近 Wi-Fi。
- 点选 SSID，输入密码后可以连接。
- 连接成功后屏幕显示 IP 地址。
- 重启后自动连接保存的 Wi-Fi。

摄像头验收：

- 屏幕 `摄像头` 页面显示实际 OV5647 画面。
- PC 后台仍能打开 `http://192.168.110.184:8080/stream.mjpg`。
- 同时预览时串口无 CSI DMA 断言。

浇灌验收：

- 点击 `10秒` 后 IO 打开，并在 10 秒后自动关闭。
- 运行中可点击 `立即关闭`。
- 自定义输入 `0` 或大于 `900` 时拒绝执行。
- 重启后 IO 默认关闭。

后台兼容验收：

- API 仍能看到设备在线。
- PC 页面仍显示实际摄像头预览。
- 远程下发限时浇灌仍可执行。
