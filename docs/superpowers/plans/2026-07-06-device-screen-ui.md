# ESP32-P4 设备屏幕 UI 烧录实施计划

## 目标

把当前 ESP32-P4 固件更新为可烧录版本，开发板本地屏幕提供中文界面：

- `网络`：扫描 Wi-Fi、点选 SSID、输入密码、保存并连接。
- `摄像头`：显示 OV5647 实际画面预览，同时保留 PC 后台 MJPEG。
- `浇灌`：本地限时打开浇灌 IO，并支持立即关闭。

## 实施步骤

1. 拆出 Wi-Fi 模块
   - 从 `app_main.c` 移出 Wi-Fi 初始化、连接等待、断线安全关闭。
   - 增加 Wi-Fi 扫描和连接保存到 NVS。

2. 扩展配置和浇灌模块
   - 增加 Wi-Fi 配置保存接口。
   - 增加浇灌剩余秒数查询，供屏幕倒计时使用。

3. 增加屏幕 UI 模块
   - 使用 Waveshare BSP + LVGL 启动触摸屏。
   - 创建 `网络`、`摄像头`、`浇灌` 三页中文界面。
   - Wi-Fi 扫描和连接放到后台任务执行，避免阻塞触摸 UI。

4. 增加摄像头屏幕预览
   - 在 `nursery_camera` 增加 RGB565 帧回调。
   - UI 模块把帧缩放到屏幕预览缓冲并刷新 LVGL 图片。
   - HTTP MJPEG 继续使用现有 JPEG 编码和 8080 端口。

5. 编译、烧录和串口验证
   - 使用 ESP-IDF v5.5.2 环境执行 `idf.py build`。
   - 检测开发板串口；串口存在后执行 `idf.py -p <port> flash monitor`。
   - 通过串口日志确认屏幕、Wi-Fi、摄像头、MJPEG、浇灌模块启动正常。

## 验收标准

- 固件编译通过。
- 开发板成功烧录。
- 串口无重启循环、无 CSI DMA assert。
- 屏幕显示中文三页界面。
- 设备摄像头页显示实际画面，PC 后台 MJPEG 继续可用。
- 本地和远程浇灌仍只能限时开启，异常默认关闭。
